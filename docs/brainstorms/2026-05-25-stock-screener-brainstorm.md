# Weekly Movers — US Equity Screener

## What We're Building

A public GitHub Pages site that screens US-listed equities (NYSE + NASDAQ)
for two volatility buckets every trading-day morning before market open:

- **Bucket A — Mid-priced movers:** stocks priced $300–$800 with weekly price
  variation ≥ $15 (absolute).
- **Bucket B — Penny movers:** stocks priced $0.79–$20 with weekly price
  variation ≥ 20% (relative).

Each screened ticker gets a per-stock dashboard page with a toggleable price
chart (1M / 6M / 1Y) and the three variation metrics. A daily narrative note,
generated by a Claude routine, summarizes what's notable in today's list and
what changed versus yesterday.

## Why This Approach

Two pipelines, separated by concern:

1. **GitHub Actions cron** (free, native, deterministic) handles data refresh
   and site generation. This is the textbook tool for a daily static-site
   refresh and lives next to the repo.
2. **Claude agents cloud routine** handles only the narrative layer, where
   LLM judgment adds real value. Keeping deterministic data work out of the
   LLM pipeline avoids paying per-run for work a Python script does better.

For data, `yfinance` covers NYSE + NASDAQ listed equities cleanly with no API
key. The $0.79 floor on Bucket B is well above true OTC pink-sheet territory,
so listed-only coverage is sufficient.

For charts, TradingView's Lightweight Charts library (MIT, ~45KB) gives
professional candlestick UI without the weight of Plotly.

## Key Decisions

- **Variation is computed three ways** (range high-low, 5-day close-to-close,
  max single-day move) and shown side-by-side. A ticker qualifies if it
  clears the threshold under **any** of the three definitions. The site
  footer explains each formula for readers.
- **Trailing 5 trading days** (rolling), not "last calendar week" — a daily
  refresh should compare to 5 sessions ago, not a stale Friday close.
- **Volume floor: $1M average daily dollar volume** on both buckets. Without
  this, Bucket B fills with pump-and-dump pink-sheet names.
- **Per-stock detail pages regenerate only for tickers in today's screen**,
  not the full universe. ~50–300 pages/day, ~80KB each.
- **Chart timeframes are toggleable** (1M / 6M / 1Y) via buttons in the UI;
  Bucket B charts default to log y-axis so percentage moves read naturally.
- **`yfinance` `auto_adjust=True`** handles splits correctly — adjusted prices
  prevent fake variation spikes after split events.
- **Universe source**: NASDAQ Trader files (`nasdaqlisted.txt` +
  `otherlisted.txt`), refreshed weekly. ~8000 tickers.
- **Cron timing**: `cron: '0 11 * * 1-5'` (UTC) for the data refresh ≈ 7am ET.
  GH Actions doesn't honor timezones, so DST causes a 1-hour drift twice a
  year. Acceptable; can split into two crons with a guard if it bites.

## Three Variation Definitions

For trailing 5 trading days:

| Name | Bucket A threshold | Bucket B threshold | What it catches |
|---|---|---|---|
| Range | max(High) − min(Low) ≥ $15 | (max(High) / min(Low) − 1) ≥ 20% | Choppy / two-sided volatility |
| 5-day return | \|Close[t] − Close[t-5]\| ≥ $15 | \|pct change\| ≥ 20% | Directional trend |
| Max daily move | max over 5 days of \|daily Δ\| ≥ $15 | max \|daily pct Δ\| ≥ 20% | Event-driven spike |

## Architecture

```
GitHub Actions cron (Mon-Fri, ~7am ET)
  ├─ fetch_universe.py     (weekly)   NASDAQ Trader → data/universe.csv
  ├─ fetch_prices.py                   yfinance batch → in-memory OHLC
  ├─ compute_screens.py                price band + volume + 3 variations
  ├─ generate_site.py                  docs/index.html + docs/stocks/<T>.html + docs/data/<T>.json
  └─ git commit + push                 GH Pages serves docs/

Claude routine (~8am ET, after data refresh commits)
  └─ generate_narrative.py             read today's + yesterday's run CSVs
                                       → docs/notes/<DATE>.html
                                       (notable today + diff vs yesterday)
```

## Per-Stock Dashboard

- Header: ticker, name, sector, last close, day change
- Three variation cards, each showing the computed value and whether it
  triggers
- Candlestick chart with volume bars; toggleable 1M / 6M / 1Y; last 5 sessions
  shaded; log y-axis for Bucket B
- Stats: market cap, avg dollar volume, sector
- Outbound links: Yahoo Finance, SEC EDGAR, Google News

## Open Questions for Planning Phase

- **DST handling**: stick with single 11:00 UTC cron and accept the 1-hour
  drift, or run two crons (11 and 12 UTC) with a date guard?
- **Persistence of detail pages**: regenerate-only-today's-screen is simpler;
  do we also want detail pages to persist (with a "last seen on" badge) so
  links from old daily notes still resolve?
- **Holiday calendar**: GH Actions cron will fire on US market holidays. Add
  a guard that skips publishing on `pandas_market_calendars` non-trading days,
  or just publish a stale-data run?
- **Repo visibility**: confirm public repo named `weekly-movers` on GitHub
  under shehral.
- **Claude routine secrets**: how the Claude routine gets a GH push token —
  fine-grained PAT in routine env, or open a PR instead of direct commit?
